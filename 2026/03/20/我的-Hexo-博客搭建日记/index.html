我的 Hexo 博客搭建日记

Author: purple&0 Date: 三月 20, 2026  16:30:00 Category: 博客搭建

一、缘起

作为计算机专业的大一新生，一直想自己做一个与计算机专业有关的作品。学院的实验室扩招计划是我的博客诞生的一个契机，于是我开始了解 Hexo + GitHub Pages 并以此来搭建我的个人博客，动手打造自己的技术博客。

二、核心步骤

1. 环境准备

  • 安装 Node.js 和 Git，为 Hexo 运行提供基础环境
  • 全局安装 Hexo 脚手架：npm install -g hexo-cli
  • 初始化博客项目：hexo init purple-blog

2. 主题与配置

  • 选用 Chic 主题，替换默认主题，让界面更简洁优雅
  • 修改 _config.yml 配置网站信息：
    • 标题：purple&0's Blog
    • 语言：zh-CN
    • 作者：purple&0
  • 配置 GitHub 部署，实现一键推送上线

3. 功能与美化

  • Favicon 替换：将 Hello Kitty 图标设为网站图标，解决浏览器缓存问题
  • 分类与标签：创建 category 和 tag 页面，实现文章分类管理
  • 特效添加：深色模式为雪花飘落，浅色模式为蜘蛛网缠绕，实现雪花飘落和蜘蛛网连线动画，让页面更生动，富有科技感

4. 部署上线

执行 hexo clean && hexo generate && hexo deploy，将博客推送到 GitHub Pages，正式上线！

三、踩坑与收获

  • Favicon 不更新：浏览器强缓存导致旧图标残留，通过覆盖 public 文件 + 无痕模式验证解决
  • 代码乱码：粘贴 JS 特效时带入不可见字符，通过重写整行代码修复语法错误
  • 部署问题：从 HTTPS 改为 SSH 连接 GitHub，解决部署失败问题

通过这次搭建，我掌握了 Hexo 博客的完整流程，理解了前端资源引入、浏览器缓存等基础概念，也体会到了「写代码 → 踩坑 → 调试 → 解决」的完整开发体验。

四、未来规划

  • 持续更新技术文章，记录计算机专业学习历程
  • 优化博客样式，添加更多个性化功能
  • 探索更多前端特效，让博客更有趣
  • 接入评论系统，与读者互动交流

这是我的第一篇博客，也是技术成长的起点。愿这个小站能成为我记录思考、分享知识的港湾 ✨
